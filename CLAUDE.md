# 데이터 모니터링 Tool PoC — CLAUDE.md

이 디렉토리는 **저장된 데이터를 콘솔에서 실시간으로 조회하는 관리자 도구** PoC다.
독립된 Git Repository로 제출한다 (`DataMonitor-영문이름-사번`).
전체 배경/도메인 모델은 상위 `../CLAUDE.md`를 참고.

## 목표

- `Json/` PoC에서 정한 저장 포맷(파일 경로/스키마)을 그대로 읽어, 별도의 콘솔
  도구에서 현재 상태를 조회할 수 있게 한다. (같은 프로세스일 필요는 없음 —
  `Main/` 애플리케이션이 쓴 데이터 파일을 이 도구가 별도 실행으로 읽어도 된다.)
- 최종 시스템의 "모니터링" 메뉴와 동일한 두 축을 제공한다.
  - **주문량 확인**: 상태별(RESERVED/CONFIRMED/PRODUCING/RELEASE) 주문 수.
    REJECTED는 무시.
  - **재고량 확인**: 시료별 현재 재고, 주문 대비 상태 표기 (여유/부족/고갈).
- "실시간"의 의미: 주기적 폴링(예: N초마다 파일 재로딩 후 화면 갱신)이든,
  사용자가 갱신 명령을 입력할 때마다 다시 읽는 방식이든 자유. 단, 재실행 없이
  최신 데이터를 다시 조회할 수 있는 갱신 동작이 반드시 있어야 한다.

## 설계 방향

- `Json/`의 Repository/파서 로직을 재사용하거나 동일한 인터페이스로 재구현한다
  (코드를 직접 import할 수 없으므로, 이 PoC 안에 필요한 최소 리더 로직을 둔다).
- 이 도구는 데이터를 **읽기 전용**으로 다룬다. 상태를 변경하는 기능(주문 승인,
  출고 처리 등)은 이 PoC의 책임이 아니다.
- 여유/부족/고갈 판정 기준은 상위 `../CLAUDE.md`의 도메인 모델과 동일하게 정의하고
  테스트로 고정한다 (예: 재고 0 → 고갈, 재고 < 주문 합계 → 부족, 그 외 → 여유).

## 기술 스택

- C++20, Visual Studio(.vcxproj), x64/x86
- GoogleTest는 NuGet으로 설치
- 데이터 파일 포맷은 `Json/`과 동일한 JSON

## TDD 진행 방식

1. 상태별 주문 수 집계 함수를 순수 함수로 만들고, 다양한 상태 조합에 대해
   gtest로 집계 결과를 검증한다 (REJECTED 제외 여부 포함).
2. 재고 상태 판정(여유/부족/고갈) 함수를 순수 함수로 만들고 경계값(정확히 0,
   정확히 주문량과 같을 때 등)을 테스트한다.
3. 파일에서 읽어와 위 함수들에 데이터를 넘기는 조립 로직을 테스트한다
   (필요 시 테스트용 임시 JSON 파일 fixture 사용).
4. 콘솔 렌더링은 최소한으로 두되, 판정/집계 로직과는 분리되어 테스트 가능해야 한다.

## 범위 밖 (다른 PoC의 책임)

- 데이터 생성/변경은 `Json/`(영속성) 및 `Dummy/`(더미 데이터)의 책임.
- 이 PoC에서 검증한 집계/판정 로직을 `Main/`의 "모니터링" 메뉴에 통합한다.
